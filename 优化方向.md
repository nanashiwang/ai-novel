## 1. 现状摘要

经过 99 个提交（约 5 周）的推进，项目已经从"有完整前端壳但无真实业务链路"升级为：

**端到端可运行的 AI 小说生产平台**，包含 6 层分层生成内核（StoryBible → Outline → ScenePlan → WriteScene → AuditScene → RewriteScene）、支持工作流（导出、MinIO、Prometheus、Admin、Job 重试 / 防重提 / 跨租户审计）、以及若干计划外的体验级能力（AI 设定共创、富文本编辑、前端架构重构）。

所有 Sprint 1–6 的 P0 闭环已落地：

```
创建项目
  → 故事启动中心（创作偏好 / 方向预览 / preflight 检查）
  → StoryBible 生成（novel_specs + characters + world_items + plot_threads）
  → 章节大纲生成（chapters；支持 batch + append + force_regenerate）
  → 场景拆分（scenes，按 chapter）
  → 场景写作（draft_versions，含 ContextBuilder、autosave、diff、版本链）
  → 审稿（continuity_issues）
  → 重写（基于 issue 修复）
  → 导出（Markdown / TXT，HTTP 流式下载，MinIO 抽象）
```

整条链路：
- 每一步创建 `generation_jobs` 并预留 quota（Free/Starter 解锁 write_scene）
- 每次模型调用写入 `model_calls`（含 prompt_key / version / 真实 cost）
- 失败 / 取消时自动 release quota + revert project status
- ModelGateway 默认走真实模型（mock 模式已完全移除，f06324c）
- OpenAI provider 支持 stream=true 兼容中转网关（f9a3079）

## 2. 架构现状

### 2.1 后端

- **API 路由**：`backend/app/api/`，按资源分模块（projects / project_extra / admin / revisions / ...）；所有受保护路由统一走 `Auth → Tenant Resolver → Permission → Entitlement → Quota → Handler → Audit Logger`
- **服务层**：`backend/app/services/`，已实现 25 个服务模块（auditor / writer / rewriter / novel_planner / context_builder / model_gateway / quota / entitlement / preflight / story_direction / story_pipeline / exporter / storage / system_settings / ...）
- **Workflow 层**：`backend/app/workflows/`，每个生成阶段独立 workflow 文件（`generate_bible / generate_outline / generate_scene_plan / write_scene / write_chapter / audit_scene / rewrite_scene / generate_full_novel`）+ `activities.py` 集中持有所有 activity；显式 `RetryPolicy`、quota release、project status revert 都已落地
- **数据库**：13 个 Alembic 迁移（`0001_initial_schema` → `0013_revision_copilot`），含复合索引（69bd6f1）、job dedupe_key（d12cfc2）、draft_versions.content_format（0012）、revision copilot 表（0013）；运行时 `schema.py` 做 idempotent `ADD COLUMN IF NOT EXISTS` 兜底
- **测试**：35 个 pytest 测试文件，覆盖 contract、auth、admin（plans/orgs/settings）、bible / outline / scene_plan / write_scene / audit_rewrite / export / dedupe / quota race（含 postgres-only）/ retry / refresh / preflight / model_gateway / markdown_stripper / revision_copilot 等

### 2.2 前端

- **路由层**：`app/`，Next.js 15 App Router；所有 page 是 thin wrapper（`import Page; export default Page`）
- **页面层**：`components/screens/`，按业务域拆分；**项目内 8 个 Page + Admin 10 个 Page 全部独立成文件**（重构于 d16bb94 / 6e9d29f）：
  - `screens/project/pages/{bible, characters, world, outline, writing, jobs, versions, export}/`
  - `screens/admin/pages/{dashboard, users, organizations, plans, quotas, generation-jobs, model-calls, content-review, settings, audit-logs}/`
- **共享原子**：
  - `components/ui/`：button / card / badge / data-table / progress / modal / form-field / admin-title / scene-editor / editor-toolbar / markdown / diff-view / stat-card / action-card / workflow-steps
  - `components/screens/project/shared/`：bible-block / preflight-card / severity / version-label / task-type-label
  - `components/screens/admin/shared/`：admin-jobs-table / types
- **Hooks**：`lib/hooks/use-admin-jobs.ts` / `use-admin-audit-logs.ts`（收敛 admin queryKey）；`pages/writing/use-{scene-versions,scene-jobs,audit-rewrite}.ts`（WritingWorkspace 内部状态）

## 5. 关键技术决策回顾

| 决策点 | 选择 | 原因 |
|---|---|---|
| Job 状态机 | `created / quota_reserved / queued / running / succeeded / failed / cancelled` | 与 Temporal 内部状态分离，便于 UI 与权限校验 |
| Job 命名 | `generate_bible / generate_outline / generate_scene_plan / write_scene / audit_scene / rewrite_scene / full_novel / export_novel` | Sprint 1 冻结，含合同测试（test_contract_consistency） |
| ModelGateway 模式 | 默认走真实 provider；mock 已完全移除（f06324c） | 避免 mock/prod 行为分裂；通过 admin 配置切换 OpenAI/Anthropic + base_url |
| Quota 预留/结算 | reserve at job create → settle / release on job end → revert project status on failure | 24c42f1 + 5e922c9 + e79a0c3 |
| Workflow Retry | 显式 `RetryPolicy` per activity；业务异常标记 non-retryable（116ae63） | 区分系统错误（重试）vs 业务错误（直接失败） |
| Outline 章节上限 | 单次按 60 章分批；项目级上限 200 | 避免 LLM 输出爆掉；append 模式只生成缺失尾部 |
| 富文本格式 | content TEXT + content_format 列；选 markdown 而非 jsonb | 后端无需新解析依赖；导出 markdown 零转换；旧 'text' 数据零侵入 |
| 编辑器 markdown serializer | 自实现（基于 ProseMirror doc） | tiptap-markdown 第三方包年久失修；StarterKit 节点集固定，自写约 80 行可控 |
| AI 设定共创安全模型 | AI 仅出"修改提案"，用户应用才写库 | 避免对话直接覆盖生产数据；保留人工最终决策权 |
| 前端聚合文件 | 全部拆为独立 Page + 子组件 | SRP；改动冲突减少；tree-shaking 生效；类型从 lib/api 统一 |
| 动态人物画像 | 单表 `character_revisions` 统一三种来源（手动 / Copilot / AI 推演） | 简化数据模型；AI 推演产出 pending 待审核，不直接覆盖；ContextBuilder 仅读 applied，避免污染 |
| 人物字段分阶段生成 | Bible 阶段只给 motivation/arc/secret 的 v0 草稿；Outline 后基于三幕重写 | 早期缺材料强行生成秘密/弧光质量差；推迟到 outline 后能引用具体章节 evidence |

## 7. 风险与缓解

原 v0 文档列的风险大多已通过 Sprint 1–6 落地缓解。本节只保留**仍有效**的风险：

| 风险 | 当前状态 | 缓解建议 |
|---|---|---|
| Workflow history 在全书生成时膨胀 | 单 scene workflow 已小粒度；全书 orchestrator 未实现 | 实现时强制 continue-as-new；按章 child workflow |
| LLM 输出 markdown 不规范（混 HTML / 代码围栏） | prompt 已禁止，但极端情况未兜底 | 解析失败时 try/except 降级为 'text'；监控 prompt key 失败率 |
| Memory / Lorebook 召回触发 prompt injection | trusted / untrusted 区分已设计但召回端未严格执行 | Sprint 候选：context_builder.v2 加来源标签 + 召回内容白名单 |
| 直接复制 AGPL 项目代码进入闭源 | 当前所有第三方代码为 MIT；自实现 markdown serializer 避免了 tiptap-markdown 这种 MIT-but-stale 包 | 继续保持；任何引入第三方需查 LICENSE |
| 支付未接入与套餐承诺不一致 | 仍未接入；后台可手动套餐切换 + 跨组织额度调整 | 接入前明确 beta 策略 + 显示"未计费" |
| Free 计划接 write_scene 后用量飙升 | preflight 已检查 entitlement + quota；Free 月配额 50000 字 | 监控 `monthly_generated_words` 接近上限的组织；自动 toast 引导升级 |
| 内容审核未接入 Moderation API | content_review 表已存在但内容仍是占位 | P1 候选 |
| 部署网络问题（国内拉 Docker Hub 慢） | Dockerfile 已用 npmmirror；后端 pip 未走镜像 | 加 pip mirror（pypi.tuna.tsinghua.edu.cn）；或文档说明 Docker Desktop 配 mirror |
